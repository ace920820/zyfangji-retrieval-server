<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment template -> deployment runtime | Operators copy `.env.example` into real deployment settings. |
| Docker Compose -> local network/services | API and Qdrant run as separate services with persistent volumes. |
| Provider config -> external model services | Embedding/reranker endpoints and API keys may cross external provider boundaries. |
| Runbook -> operations team | Commands and privacy expectations become operational behavior. |

## STRIDE/Safety Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Information Disclosure | `.env.example` | mitigate | Use empty API key placeholders and comments warning not to commit real credentials. |
| T-05-03-02 | Denial of Service | Docker Compose | mitigate | Package bounded demo services only; avoid load generator behavior and keep latency script separate. |
| T-05-03-03 | Information Disclosure | provider configuration docs | mitigate | Document provider boundary and logging rule forbidding raw patient text/API keys/full provider error bodies. |
| T-05-03-04 | Safety/Misuse | deployment docs | mitigate | Repeat retrieval-only, no diagnosis/prescribing, no chat/admin/NER/MySQL v1 boundaries. |
| T-05-03-05 | Repudiation | runbook validation | mitigate | Include explicit smoke, status, health, latency, and manual UAT commands with expected use. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Smoke fixture -> demo client | Checked-in sample patient-query data becomes reviewer-visible output. |
| Demo client -> live API | Patient query payload crosses the network only when live mode is explicitly selected. |
| Demo output -> reviewer interpretation | Formula/evidence/ranking output can be mistaken for clinical advice if labels are unsafe. |
| README/demo guide -> operator workflow | Commands will be copied into demo environments. |

## STRIDE/Safety Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Information Disclosure | `scripts/demo_smoke.py` live mode | mitigate | Default offline mode sends no network traffic; live mode requires explicit `--mode live --base-url`. |
| T-05-02-02 | Safety/Misuse | demo output | mitigate | Output labels must use formula/evidence/warnings/score semantics and avoid diagnosis/advice/prescribing/confidence labels. |
| T-05-02-03 | Repudiation | demo results | mitigate | JSON output includes mode, query_count, result summaries, and score semantics for reproducible review. |
| T-05-02-04 | Denial of Service | demo client | mitigate | Sequential fixture execution with bounded `--limit`; no concurrent load generation. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Search service -> SQLite index state | Search reads active resource pointers and must not mutate lifecycle records. |
| Search service -> Qdrant client | Query vector and active collection cross into vector store infrastructure. |
| Search service -> reranker provider | Patient query text and candidate passages cross to a model/provider boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | active index resolution | mitigate | Resolve Qdrant collection and BM25 path from `SQLiteIndexStateStore.get_active()` only; never from alias alone. |
| T-03-02-02 | Denial of Service | recall/rerank pipeline | mitigate | Use `settings.recall_topk = 50`, request `topk <= 50`, and rerank at most fused Top50 candidates. |
| T-03-02-03 | Information Disclosure | reranker/provider calls | mitigate | `BGERerankerProvider` must not log raw query text or candidate evidence; deterministic provider is restricted to tests/dev fallback. |
| T-03-02-04 | Repudiation | provider/model outage behavior | mitigate | Raise `reranker_unavailable` when reranker is required; otherwise include `reranker_degraded` warning and pipeline status. |
| T-03-02-05 | Spoofing | stale/partial index state | mitigate | Treat missing active index, missing BM25 path, or mismatched active counts as `index_not_ready` before returning results. |
| T-03-02-06 | Safety/Misuse | fused/rerank scores | mitigate | Preserve signal scores as diagnostics and leave medical-confidence wording out of candidate and response models. |
</threat_model>

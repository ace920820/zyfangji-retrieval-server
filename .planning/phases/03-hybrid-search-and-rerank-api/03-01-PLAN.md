<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Java backend -> POST /api/search | Structured patient presentation data enters the retrieval service. |
| FastAPI validation -> search dependency | Untrusted JSON is converted into typed models and either dispatched or rejected. |
| OpenAPI/schema -> Java clients | Score and error semantics become external integration contracts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Information Disclosure | `api/routes/search.py` | mitigate | Do not log request bodies; task actions add no logging of patient fields. |
| T-03-01-02 | Denial of Service | `PatientSearchRequest` | mitigate | Bound `topk` to `1..50`, `symptoms` to max 20 items, symptom item strings to 200 chars, and `main_symptom`/`tongue`/`pulse`/`syndrome` strings to 500 chars through Pydantic validation. |
| T-03-01-03 | Repudiation | error responses | mitigate | Return `SearchErrorEnvelope` with stable `error.code`, `error.message`, and `error.details`. |
| T-03-01-04 | Safety/Misuse | score schema | mitigate | Require `score_semantics` text stating scores are not medical confidence, diagnosis probability, or prescription certainty. |
| T-03-01-05 | Tampering | route registration | mitigate | Register only read-only status routes and one stateless POST search route; no import/rebuild mutation endpoint is added in Phase 3. |
</threat_model>

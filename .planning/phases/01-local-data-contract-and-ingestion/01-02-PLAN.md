<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local `.xlsx` file -> pandas/openpyxl parser | Workbook content is untrusted input even when local. |
| Parsed row dict -> canonical `KnowledgeEntry` | Source text becomes normalized retrieval metadata. |
| CLI output -> terminal/log capture | Import output can be copied into logs and must avoid raw clinical-like row dumps. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | `excel_reader.py` | mitigate | Require exact 22-header manifest and raise `ValueError` on header mismatch before mapping rows. |
| T-01-06 | Tampering | `mapper.py` | mitigate | Preserve full `raw_record` and explicit row issues so formula/evidence data corruption is detectable. |
| T-01-07 | Information Disclosure | `cli.py` | mitigate | Print counts, row numbers, and issue codes only; do not print full raw row contents in import reports. |
| T-01-08 | Denial of Service | workbook parsing | accept | Phase 1 reads a known 1,329-row local workbook; large-file limits can be added when HTTP upload/import exists. |
| T-01-09 | Elevation of Privilege | file path handling | mitigate | CLI accepts local `Path` only and does not fetch URLs or execute workbook macros. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local repo -> Python runtime | Package configuration controls what code and dependencies execute during ingestion. |
| Workbook row text -> canonical model | Untrusted workbook cell text becomes structured records and later search metadata. |
| Formula text -> structured formula mentions | Ambiguous clinical branch text crosses into machine-readable fields. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `src/zyfangji_retrieval/domain/ids.py` | mitigate | Generate `entry_id` from explicit stable content parts with SHA-256 and never from sparse source `编码`. |
| T-01-02 | Tampering | `src/zyfangji_retrieval/ingestion/formulas.py` | mitigate | Preserve `formula_raw` and mark branch-heavy text `needs_review` so ambiguous evidence is not silently corrupted. |
| T-01-03 | Information Disclosure | future logging callers of these contracts | mitigate | Keep Phase 1 model helpers pure; no raw row logging in contract helpers. Later import reports must log row numbers/counts rather than raw clinical-like text. |
| T-01-04 | Spoofing/Tampering | `pyproject.toml` dependencies | mitigate | Pin exact dependency versions from research to avoid unreviewed parser/runtime drift in the demo. |
</threat_model>

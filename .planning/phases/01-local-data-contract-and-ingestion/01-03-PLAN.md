<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Canonical entries -> SQLite/JSONL | Raw medical-adjacent source text is persisted for audit. |
| SQLite metadata -> future index rebuild | Local metadata becomes the trusted source for Qdrant/BM25 in Phase 2. |
| CLI file path -> local filesystem | Operator-provided DB/export paths can write files on disk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Tampering | `persistence/sqlite.py` | mitigate | Save import batch, raw rows, normalized entries, and row issues in one SQLite transaction. |
| T-01-11 | Tampering | `ingestion/importer.py` | mitigate | Use deterministic `entry_id` from Plan 01 and verify repeated imports preserve the same ID set. |
| T-01-12 | Information Disclosure | `cli.py` | mitigate | CLI prints report counts and rebuild counts only; raw records remain in metadata files, not terminal logs. |
| T-01-13 | Tampering | `persistence/jsonl.py` | mitigate | JSONL export uses Pydantic serialization and one object per line so audit snapshots can be diffed and reloaded. |
| T-01-14 | Spoofing/Tampering | customer MySQL boundary | mitigate | Phase 1 rebuild source is only SQLite/JSONL local metadata; tests grep for absent MySQL/SQLAlchemy dependencies. |
| T-01-15 | Tampering | CLI `--db-path` and `--jsonl-export` | accept | Phase 1 CLI is local operator tooling; restrict HTTP import path handling in later API phases before exposing remote callers. |
</threat_model>

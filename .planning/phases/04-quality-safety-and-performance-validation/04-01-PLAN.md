<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Real workbook -> regression tests | Source Excel content is transformed into canonical entries and can create false confidence if tests only inspect mocks. |
| Patient text -> fake provider failures | Test patient text crosses fake provider/vector/reranker boundaries and must not appear in outward errors. |
| Search response -> Java/demo consumers | Response shape can be misread as diagnosis or prescription if unsafe fields or score wording slip in. |
| Active index state -> search service | Missing or stale index records must stop search instead of returning partial or stale results. |

## STRIDE/Safety Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `tests/test_quality_regression_contract.py` | mitigate | Use the real sample workbook path plus existing ingestion mapper to prevent mock-only false positives for QUAL-01. |
| T-04-01-02 | Information Disclosure | provider failure tests | mitigate | Assert `隐私患者文本` is absent from `SearchServiceError` string, message, and details for embedding/vector/reranker failures. |
| T-04-01-03 | Safety/Misuse | serialized `SearchResponse` | mitigate | Recursively ban `diagnosis`, `diagnosis_probability`, `prescription`, `prescription_certainty`, `medical_advice`, `autonomous_diagnosis`, `autonomous_prescription`, `treatment_plan`, and `confidence` keys. |
| T-04-01-04 | Safety/Misuse | evidence projection | mitigate | Assert `source`, `formula_raw`, `formula_mapping_status`, `evidence.contraindication`, and `evidence.western_medicine_priority` are visible when source data contains them. |
| T-04-01-05 | Spoofing | active index readiness | mitigate | Test missing/empty active index raises `index_not_ready` before any results are returned. |
| T-04-01-06 | Repudiation | test coverage claims | mitigate | Keep the tests cross-cutting and reference Phase 1-3 validation docs; do not claim live provider/Qdrant quality is proven by offline fakes. |

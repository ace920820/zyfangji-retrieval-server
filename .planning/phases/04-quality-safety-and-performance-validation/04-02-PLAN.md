<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Smoke fixture -> test runner | Fixture expectations can become brittle or overclaim medical quality if they freeze exact scores or ranks. |
| Patient query payload -> live latency mode | Structured patient text crosses to a configured HTTP service only when explicitly opted in. |
| Latency report -> demo stakeholders | P50/P95 numbers can be misinterpreted as medical quality or as including import/model-build time. |
| Offline fakes -> validation claims | Offline deterministic tests prove contract stability, not live provider ranking quality. |

## STRIDE/Safety Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | `tests/fixtures/smoke_queries.json` | mitigate | Store category, request, min result count, must-have field paths, banned fields, and top-N formula anchors; avoid exact raw score snapshots. |
| T-04-02-02 | Safety/Misuse | `tests/test_smoke_queries.py` | mitigate | Recursively ban generated diagnosis/advice/prescribing/confidence keys and require evidence/contraindication paths in smoke responses. |
| T-04-02-03 | Information Disclosure | `scripts/search_latency.py` live mode | mitigate | Keep live mode opt-in via `--mode live --base-url`; default offline mode sends no patient text to network services. |
| T-04-02-04 | Repudiation | latency JSON | mitigate | Report `mode`, `query_count`, `sample_count`, thresholds, dataset/index metadata when available, and `thresholds_passed` so results are auditable. |
| T-04-02-05 | Denial of Service | latency harness | mitigate | Default to bounded `--runs 20`, `--warmups 3`, fixture TopK bounds, and no concurrent load generator behavior. |
| T-04-02-06 | Safety/Misuse | P50/P95 interpretation | mitigate | Script and plan scope state latency measures indexed `/api/search` only and excludes import/corpus embedding; do not present latency as retrieval quality or medical correctness. |
</threat_model>
